# Accessing RCC clusters

To connect to RCC resources, you must have an RCC user account sponsored by PI ([request an account](https://rcc.uchicago.edu/accounts-allocations/request-account){:target='_blank'}). The RCC account uses your UChicago CNetID and its corresponding password. All users are responsible for knowing and abiding by the [RCC Terms of Use](https://rcc.uchicago.edu/about-rcc/rcc-user-policy).

## Supported protocols
The following table provides a summary of supported connection methods:

|  <div style="width:150px">Connection Method | Description | GUI Support | Access to Compute Nodes |   
| ------------------------------------------- | ----------- | ------- | ------- | 
| [SSH, SCP, SFTP](ssh/main.md)  | A command line interface | Limited | Yes (tunneling) |
| [Open OnDemand](open_ondemand/open_ondemand.md) | A user-friendly web-based portal  | Full-scale  | Yes (direct) 
| [ThinLinc](thinlinc/main.md) | A web-based or client-based desktop  | Full-scale | Yes (tunneling) | 
|[SAMBA](../data_transfer/persistent_mapping/samba.md)| A persistent network drive to sync local data with remote | File Manager | No |
|[Globus](globus/access-files.md)| A web-based platform for asynchronous data transfer, sharing, and management | File Manager | No |
